# 教材管理网站项目说明

## 项目简介
本项目是一套基于Django框架的教材管理网站，采用前后端分离架构，旨在通过数字化手段优化教材管理全流程，提升学校教材管理的便捷性与高效性。系统包含前台展示和后台管理功能，可满足管理员、教师、学生三类用户的不同需求。

## 技术栈
### 开发语言与框架
- **Python**：作为开发语言，其简洁易读、可扩展性强的特点适配项目需求。
- **Django框架**：用于搭建Web架构，利用其MTV模式（模型、模板、视图）实现业务逻辑与数据展示的解耦。
- **Django REST framework**：用于构建RESTful API，支持前后端数据交互。

### 数据库
- **MySQL**：凭借其运行速度快、安全性高、代码简洁的优势，保障数据存储与处理的高效性，数据共享性高、冗余度低且容易扩充。

### 前端技术
- **Vue2.0**：用于构建用户界面的渐进式JavaScript框架。
- **Element-ui**：基于Vue2.0的桌面端组件库。
- **Vue Router**：用于前端路由管理。
- **Axios**：用于发送HTTP请求。
- **ECharts**：用于数据可视化展示。
- **Sass**：用于样式编写。

### 开发环境
- **PyCharm**：用于后端Python/Django开发，提升编码与调试效率。
- **VSCode**：用于前端Vue开发。

### 架构模式
- **B/S架构**（Browser/Server,浏览器/服务器）：确保用户通过浏览器即可便捷访问，将系统功能的关键部分集中在服务器上，简化系统的开发、维修及利用。

## 运行环境要求
### 软件环境
- Python：3.7版本
- Node.js：14版本
- MySQL：5.7版本
- 数据库管理工具：Navicat10+

## 项目结构
### 后端结构
- `django47nia/dj2`：Django项目主目录
  - `views.py`：视图处理函数，负责业务逻辑。
  - `urls.py`：URL路由配置，将页面请求分发给不同的View处理。
  - `settings.py`：项目配置。
- `main`：主要应用模块，包含系统核心功能的实现代码。

### 前端结构
前端代码位于`项目名称/templates`目录下
- `front/admin`：后台管理系统前端代码
- `src/components`：通用组件目录
- `src/views`：页面视图目录
- `src/router`：路由配置目录
- `src/utils`：工具函数目录
- `src/assets`：静态资源目录（css、img等）

## 部署与运行步骤
### 后端部署
1. 克隆项目到本地
   ```bash
   git clone https://github.com/Zhouyuhong-123/work2.git
   cd work2/django47nia
   ```
2. 安装依赖包
   ```bash
   pip install -r requirements.txt
   ```
3. 配置数据库
   修改`config.ini`文件中的数据库配置信息，确保MySQL5.7服务已启动。
4. 启动Django服务
   ```bash
   python manage.py runserver 8080
   ```

### 前端部署
1. 进入前端目录
   ```bash
   cd templates/front/admin
   ```
2. 安装依赖
   ```bash
   npm install
   ```
3. 运行开发环境
   ```bash
   npm run serve
   ```
4. 构建生产环境
   ```bash
   npm run build
   ```

## 主要功能模块
### 管理员模块
负责全系统数据管理，包括：
- 学生管理：对学生信息（学号、姓名、头像、性别、手机号码、邮箱等）进行增删改查。
- 教师管理：对教师信息（教师工号、姓名、头像、性别、联系电话、邮箱等）进行增删改查。
- 供应商管理：对供应商信息（负责人、联系电话、公司名称、公司地址、添加日期等）进行增删改查。
- 教材类型管理：对教材类型信息进行查询、新增或删除等操作。
- 教材信息管理：对教材信息（教材编号、名称、封面、类型、数量、单价、供应商、添加日期等）进行增删改查及入库操作。
- 教材入库管理：对教材入库信息（入库编号、教材编号、名称、类型、数量、单价、合计、入库日期、供应商、备注等）进行查询或删除。
- 领书与退书管理：对教师和学生的领书、退书信息进行查询或删除等操作。

### 学生模块
- 登录与注册：通过学号等信息完成注册和登录。
- 教材查询：查询所需教材的相关信息。
- 领书与退书：提交领书、退书申请，查看自己的领书、退书记录。
- 个人中心：管理个人信息及修改密码。

### 教师模块
- 登录与注册：通过教师工号等信息完成注册和登录。
- 教材查询：查询所需教材的相关信息。
- 领书与退书：提交领书、退书申请，查看自己的领书、退书记录，功能适配教师角色需求。
- 个人中心：管理个人信息及修改密码。

### 其他功能
- 数据统计与可视化展示：通过ECharts实现教材相关数据的图表展示。
- 权限管控：严格的权限管理机制，确保不同角色只能操作其权限范围内的功能。

## 系统特点
- **分层架构**：分离表现层、业务逻辑层与数据访问层，降低耦合度，提升系统的可维护性和可扩展性。
- **安全性高**：对用户密码进行加密处理，通过权限管控确保数据安全，用户操作信息会被监控。
- **操作便捷**：界面简洁直观，用户易上手，无论是管理员的全面管理操作，还是学生、教师的查询与领退书操作，都简单易懂。
- **性能可靠**：系统响应速度快，能处理一定的并发访问，代码简洁，不会占用过多计算机内存。

## 注意事项
- 开发环境中前后端通过代理进行通信，配置位于`vue.config.js`。
- 生产环境需注意静态资源路径配置。
- 数据库连接信息需根据实际环境修改。
- 系统发行后，若出现bug可在不影响使用者的情况下进行线上维护，确保系统的稳定运行。
